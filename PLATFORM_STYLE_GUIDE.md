# FrontDesk Agents - Platform Style Guide

## 1. Introduction

This style guide provides a set of standards for all text content on the FrontDesk Agents platform. Following these guidelines ensures consistency, professionalism, and a high-quality user experience.

**Last Updated:** January 7, 2026

---

## 2. Tone & Voice

Our brand voice is **professional, confident, and helpful**.

| Area | Tone | Example |
|---|---|---|
| **Marketing** | Confident, benefit-focused | "Transform Your Front Office With AI Agents" |
| **Dashboard** | Informative, action-oriented | "Welcome back! Here's your platform overview." |
| **Legal** | Clear, formal, compliant | "This Policy describes how FrontDesk Agents collects and uses information." |
| **Errors** | Helpful, non-technical | "Please enter a valid email address." |

---

## 3. Capitalization

### Product Name
- **Correct:** FrontDesk Agents
- **Incorrect:** Frontdesk Agents, Front Desk Agents, frontdesk agents

### Feature Names
- **Rule:** Use Title Case for all feature names.
- **Correct:** AI Voice Agents, Real-time Analytics, Multi-Language Support
- **Incorrect:** AI voice agents, real-time analytics, multi-language support

### Button Text
- **Rule:** Use Title Case for all button text.
- **Correct:** Start Free Trial, Schedule Demo, View Details
- **Incorrect:** start free trial, Schedule demo, view details

### Navigation Items
- **Rule:** Use Title Case for all navigation items.
- **Correct:** Dashboard, AI Agents, Pricing, Features
- **Incorrect:** dashboard, ai agents, pricing, features

### Headings
- **Rule:** Use Title Case for all H1, H2, and H3 headings.
- **Correct:** Trusted by Industry Leaders, How We Use Data
- **Incorrect:** Trusted by industry leaders, how we use data

---

## 5. Punctuation

- **Periods:** Use at the end of all complete sentences.
- **Commas:** Use serial commas (e.g., "A, B, and C").
- **Exclamation Marks:** Use sparingly and only for positive reinforcement (e.g., "Welcome!").
- **Quotation Marks:** Use double quotes for all direct quotes.

---

## 6. Writing Best Practices

### Be Clear & Concise
- **Good:** "Natural-sounding AI agents that handle calls 24/7."
- **Bad:** "Our state-of-the-art AI agents are capable of handling calls at any time of day or night with a high degree of naturalness."

### Use Active Voice
- **Good:** "The AI handles initial client screening."
- **Bad:** "Initial client screening is handled by the AI."

### Be Benefit-Focused
- **Good:** "Reduce no-shows by sending timely reminders."
- **Bad:** "The system can send reminders to patients."

### Be Consistent
- Use the same term for the same concept throughout the platform.
- Follow capitalization rules consistently.
- Maintain a consistent tone of voice.

---

## 7. Error Messages

### Principles
1. **Be helpful:** Explain what went wrong and how to fix it.
2. **Be clear:** Use plain language, not technical jargon.
3. **Be polite:** Avoid blaming the user.
4. **Be specific:** Provide context where possible.

### Examples
- **Good:** "Please enter a valid email address."
- **Bad:** "Invalid input."

- **Good:** "Password must be at least 8 characters."
- **Bad:** "Password too short."

- **Good:** "An error occurred. Please try again."
- **Bad:** "API request failed."

---

## 8. Accessibility (A11y)

- **Alt Text:** All images must have descriptive alt text.
- **Link Text:** Link text should be descriptive (e.g., "View Pricing" not "Click Here").
- **Labels:** All form inputs must have clear labels.
- **ARIA Attributes:** Use ARIA attributes to provide context for screen readers.

---

## 9. Review Process

Before committing any new text content, please ensure it meets the following criteria:

1. [ ] **Adheres to Style Guide:** Follows all rules in this document.
2. [ ] **Spell Checked:** No spelling errors.
3. [ ] **Grammar Checked:** No grammatical errors.
4. [ ] **Consistent:** Uses correct terminology and capitalization.
5. [ ] **Clear & Concise:** Easy to understand.
6. [ ] **Accessible:** Follows accessibility best practices.
